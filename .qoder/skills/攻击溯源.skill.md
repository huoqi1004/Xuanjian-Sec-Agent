# 攻击溯源 (Attack Forensics)

## 基本信息
- **ID**: attack_forensics
- **类别**: 安全事件
- **图标**: 🔎
- **版本**: 1.0.0
- **状态**: active

## 描述
对安全事件进行分析和溯源，确定攻击来源和攻击路径。通过收集和分析各种日志和证据，还原攻击过程，为安全事件响应提供支持。

## 输入参数
| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `event_id` | string | 是 | - | 安全事件ID |
| `time_range` | string | 否 | "24h" | 时间范围，如`24h`、`7d`、`30d` |
| `evidence` | string | 否 | "" | 事件证据，如日志文件路径、可疑IP等 |
| `output_format` | string | 否 | "json" | 输出格式：`json`、`report`、`forensic` |
| `include_recommendations` | boolean | 否 | true | 是否包含防御建议 |

## 输出结果
返回一个JSON对象，包含：
- `event_summary`: 事件摘要：
  - `event_id`: 事件ID
  - `timestamp`: 事件时间
  - `severity`: 事件严重程度
  - `affected_systems`: 受影响的系统
- `attack_path`: 攻击路径：
  - `initial_access`: 初始访问方式
  - `attack_chain`: 攻击链
  - `affected_resources`: 受影响的资源
- `attacker_profile`: 攻击者画像：
  - `attacker_type`: 攻击者类型
  - `tactics`: 攻击战术
  - `tools_used`: 使用的工具
  - `possible_motivation`: 可能的动机
- `evidence`: 证据列表
- `recommendations`: 防御建议

## 调用方式
1. 收集事件相关的日志和证据
2. 分析日志数据，识别攻击模式
3. 构建攻击时间线和攻击路径
4. 识别攻击者特征和可能的动机
5. 生成溯源报告和防御建议
6. 可选：与SIEM系统集成，获取更多事件上下文

## 依赖工具/API
- ELK Stack（日志分析）
- SIEM系统（安全事件管理）
- 威胁情报平台（攻击者情报）
- 本地大模型（事件分析和溯源）
